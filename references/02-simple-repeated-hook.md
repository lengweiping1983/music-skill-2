本项来源清单见 [source-bank-50.md](source-bank-50.md) S01-S50；重点使用 earworm、重复记忆、Max Martin Hook、chorus首句传播、melodic math、Hook Preview 资料。

## 共识结论

- 最强 Hook 通常是旋律、歌词、节奏或音色中至少两种叠加的 compound hook。
- Hook 字数越短越容易被字幕、评论、搜索和翻唱使用。
- 重复不是偷懒；关键是重复核心句，并让每次重复有能量、音高、和声或节奏变化。
- 标题最好进入 Chorus 首句或末句，方便用户搜索。
- Hook 首句要像真人会说会唱的话，避免“正确判断句”。

## 冲突观点

- 复杂 Hook 可能更高级，但病毒传播优先低认知负担。
- 口号式 Hook 有力量，但过空会像宣传语；要用动作、画面或身体感落地。


```markdown

```

## 反例

- 副歌每行都换新概念，听众不知道哪句是 Hook。
- Hook 首句是“我们要勇敢追梦”。
- 标题和副歌没有关系。

## 交付自检

- Hook 是否能被普通人听一遍复述？
- 标题是否出现在 Chorus？
- 核心句是否重复至少2-3次？
- 清唱时 Hook 是否仍成立？
