# 中文歌词与结构规范

本文件把中文歌词、押韵、结构呼吸和 Suno 中文发音规则压缩为本 Skill 的本地规范。写歌时吸收规则，不把分析写入 `suno.md`。

## 中文歌词核心

- 主歌写具体：时间、地点、物件、动作、台词。
- 副歌写可唱：短句、重复、韵脚、标题 Hook。
- 情绪从细节长出来，不直接堆“难过、孤独、勇敢、梦想”。
- Hook 首句要像真人会说会唱的话，不像作文标题。
- 每个核心段落至少有一句 6-12 字可做短视频字幕。

## 行长和押韵

- 中文歌词优先每行4-10字；叙事句可略长，但必须能换气。
- Chorus 优先4-6行，核心句重复2-3次。
- Verse 优先4-8行，用画面推进故事。
- 副歌尽量押同一韵，主歌和副歌可以换韵制造层次。

常用韵脚感觉：

- `a/ia/ua`：明亮、开阔，适合燃和甜。
- `ei/ui`：感伤、飘逸，适合伤感和释怀。
- `ou/iu`：悠长、缠绵，适合怀旧和失恋。
- `ang/iang/uang`：宏大、昂扬，适合 anthem 和燃曲。
- `eng/ing/ong`：高远、苍茫，适合城市夜景、国风和史诗。

## 画面优先

弱写法：

```text
我很难过
我忘不了你
我们的爱回不去
```

强写法：

```text
凌晨三点半
外卖袋还热
你那句早点睡
亮了一整夜
```

每段 Verse 至少保留一个不可替换细节。职业歌、军旅歌、底层逆袭歌尤其需要真实物件：工牌、靴带、饭盒、键盘、站牌、车票、矿灯、口袋里的纸条等。

## 结构呼吸

推荐能量弧线：

```text
Intro 声音指纹 → Hook Preview 抓耳 → Verse 具体画面 → Pre-Chorus 收紧 → Chorus 释放 → Post-Chorus 二级 Hook → Bridge 降低或换视角 → Final Chorus 最高点
```

段落对比：

- Verse：叙事、较少乐器、句子更具体。
- Pre-Chorus：情绪收紧，句子变短，能量上升。
- Chorus：旋律更简单，歌词更短，Hook 重复。
- Bridge：给新视角或不舒服但真实的揭示。
- Final Chorus：保留主 Hook，但至少有一处语义升级。

## 中文发音和 Suno 稳定性

- 避免生僻字、多音字和绕口连续字。
- 避免连续太多第四声导致硬、冲、难唱。
- 关键词放在重拍和开口音上更容易听清。
- 括号里不要写声调理论，只写可执行演唱提示，如 `(咬字清楚, 人声靠前)`。
- 如果使用方言感，只写轻度标签，如 `slight Cantonese color`，不要让方言破坏可懂度。

## 反口号规则

正能量、军旅、职业和逆袭题材最容易口号化。修正方式：

- `我们要守护山河` → `靴带湿了还没松`
- `我要重新开始` → `我把明天重新开机`
- `梦想照亮远方` → `地铁玻璃照见我`
- `安全第一最重要` → `他把刹车踩到发抖`

## 自检

- Chorus 首句是否能被普通人复述？
- Verse 是否有真实物件或动作？
- Pre-Chorus 是否给出新判断，而不是“我终于懂了”？
- Bridge 是否提供新视角？
- Final Chorus 是否升级，而不是原样复制？
